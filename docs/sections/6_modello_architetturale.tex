\section{Modello Architetturale}

\subsection{Architettura Hardware}
Il sistema A.L.I.C.E. è progettato per operare su un'infrastruttura \textbf{serverless/managed}, sfruttando Supabase come Backend-as-a-Service e Vercel per il deploy del frontend Next.js. Non è necessario gestire server fisici.

\centering
\renewcommand{\arraystretch}{1.3}
\begin{tabular}{|l|l|l|}
\hline
\textbf{Componente} & \textbf{Servizio} & \textbf{Dettaglio} \\
\hline
Frontend / SSR & Vercel & Deploy automatico da repository Git \\
\hline
Database (PostgreSQL) & Supabase & Istanza PostgreSQL managed \\
\hline
Autenticazione & Supabase Auth & Magic link, email/password \\
\hline
API REST & Supabase (PostgREST) & API auto-generate dalle tabelle \\
\hline
Storage file & Supabase Storage & Bucket per immagini e risorse \\
\hline
\end{tabular}
\caption{Infrastruttura cloud del sistema A.L.I.C.E.}
\label{tab:infra_cloud}
\end{table}

I dispositivi client accedono tramite browser web. Il paziente anziano dovrebbe disporre di un tablet da almeno 10'' o un PC desktop; l'operatore sanitario può usare PC, laptop o tablet. Non è richiesta l'installazione di software aggiuntivo.

\subsubsection{Stack Tecnologico}
\begin{itemize}
    \item \textbf{Next.js 14+} $\rightarrow$ Framework React full-stack. App Router con routing basato su file system, Server Components per ridurre il JavaScript lato client, Server Actions per le mutazioni dati.
    \item \textbf{React / React-DOM} $\rightarrow$ Libreria UI per la costruzione dei componenti.
    \item \textbf{Supabase} $\rightarrow$ Backend-as-a-Service open source basato su PostgreSQL. Fornisce database managed, API auto-generate (PostgREST), autenticazione, Row Level Security e storage file.
    \item \textbf{@supabase/supabase-js} $\rightarrow$ Client JavaScript per l'interazione con Supabase.
    \item \textbf{@supabase/ssr} $\rightarrow$ Integrazione Supabase con il rendering server-side di Next.js.
    \item \textbf{Recharts / Chart.js} $\rightarrow$ Librerie per grafici e visualizzazioni nella dashboard.
    \item \textbf{React Icons} $\rightarrow$ Set di icone vettoriali.
    \item \textbf{Zod} $\rightarrow$ Validazione e tipizzazione degli schemi dati.
    \item \textbf{CSS Modules} $\rightarrow$ Styling con scope locale per componente.
\end{itemize}

\subsubsection{DBMS}
Il database è \textbf{PostgreSQL 15}, gestito da Supabase. L'accesso avviene tramite le API auto-generate (PostgREST) per le operazioni CRUD standard, oppure tramite SQL diretto per operazioni complesse, stored procedures e policy RLS. PostgreSQL è stato scelto per la sua maturità, l'assenza di vendor lock-in e il supporto nativo a Row Level Security.

\subsubsection{Browser e Sistemi Operativi Supportati}
Essendo un'applicazione web, il sistema funziona su qualsiasi sistema operativo con un browser moderno. I browser supportati sono: Chrome 90+, Firefox 88+, Edge 90+, Safari 14+, Samsung Internet 14+. Il design responsive si adatta a schermi da 768$\times$1024 pixel (tablet) fino a risoluzioni Full HD e superiori.
