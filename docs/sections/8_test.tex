\section{Test}

I test del progetto sono stati organizzati in modo pratico, controllando sia la correttezza tecnica
della build sia i flussi principali usati da operatore e paziente.

\subsection{Strumenti utilizzati}

\begin{itemize}
    \item \textbf{Build Next.js:} il comando \texttt{npm.cmd run build} controlla compilazione, tipi TypeScript e generazione delle route.
    \item \textbf{Browser e DevTools:} usati per verificare navigazione, responsive design, richieste di rete e assenza di errori runtime.
    \item \textbf{Supabase Dashboard:} usata per controllare dati inseriti, tabelle, funzioni SQL e policy RLS.
    \item \textbf{Postman:} usato per testare manualmente le API Supabase e, quando configurata la chiave, la route del suggerimento AI.
\end{itemize}

\subsection{Verifiche sul database}

Le verifiche su Supabase hanno controllato:

\begin{itemize}
    \item creazione automatica del profilo dopo la registrazione;
    \item associazione corretta tra operatore e paziente;
    \item inserimento dei log giornalieri per farmaci, esercizi e umore;
    \item visibilita' dei dati coerente con le policy RLS.
\end{itemize}
